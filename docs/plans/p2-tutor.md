**선행 의존(사용자 액션)**: 실기기 검증 결과 반영(ADR-0003 지연 기준 갱신) — W2 착수 전 권장.

barge-in 175ms, ~$0.035–0.063/분. AI 튜터는 Realtime, 레슨은 turn-based(ADR-0007).

## 2. 작업 목록

### W1. Realtime API POC (스파이크 — 가부 판정) ✅ 완료 (2026-06-13)

- OpenAI Realtime API로 1세션 양방향 음성 대화 E2E (노드 스크립트 → 성공 시 RN 연결 검증)
- 측정: 첫 응답 지연, 끊김, 분당 비용 실측 → **turn-based 대비 판정 기준을 ADR로 기록**
- 판정 결과에 따라 W2의 전송 계층 결정 (불가/과비용이면 AI 튜터도 turn-based 폴백 — PLAN 허용)
- **완료**: ADR-0007 **승인** — gpt-realtime(GA), 첫 응답 0.63s(turn-based 3.51s 대비 ~5.6×),
  barge-in 175ms, 턴당 ~$0.006–0.010(~$0.035–0.063/분). **AI 튜터는 Realtime 채택**(W2 전송 계층),
  레슨은 turn-based 유지. 비용은 일일 시간 상한·세션 길이 cap으로 통제(W2 반영). 스파이크 `packages/ai/spike/realtime.mts`.

### W2. 프리토킹 (tutor 탭 — 스텁 → 실제)

- **전송 계층: Realtime (ADR-0007 승인)** — 서버 VAD on(자연 turn-taking) + 클라이언트 barge-in.
  일일 시간 상한·세션 길이 cap을 하드하게 적용(비용 통제). 시나리오별 토큰 프로파일 재측정으로 분당 비용 갱신.
- 주제 선택 → 시간 제한 자유 대화(기본 5분, 비용 상한) → 종료 요약(교정 집계)
- 히스토리 슬라이딩 윈도우(6턴)·턴당 30초 cap 등 레슨과 동일 제약 재사용
- tutor_sessions 스키마 (lesson_sessions 패턴 — RLS·불변 턴 로그 동일 원칙, **보안 민감**)
- **완료 기준**: 실기기에서 5분 프리토킹 완주, 일일 상한 동작

### W3. 롤플레이 시나리오 ✅ 완료 (2026-06-13, 세션 6)

- 시드 시나리오 4종(레스토랑·공항·면접·호텔) — content/ JSON 스키마 확장(zod 단일 출처),
  `validate:content` 통과
- 역할·목표·종료 조건이 있는 대화 (프리토킹과 같은 전송 계층, 프롬프트만 분기)
- **완료 기준**: 시나리오 1개 완주 시 목표 달성 판정 노출
- **완료**: W2 seam 재사용(새 테이블·전송 없이 `tutor_sessions`·`TutorTransport`에 얹음, ADR-0009).
  목표 추적은 전송→코어 `metObjectiveIds` 신호(코어 신뢰 경계). vitest 353·E2E tutor 10/10(롤플레이 4).
  작업계획서 `docs/plans/p2-w3-roleplay.md`.

### W4. 발음 점수·피드백 UI (결정 필요 — 스파이크 선행)

- **완료 기준**: Drill에서 단어별 점수 시각화, 약점 음소 1개 이상 식별
