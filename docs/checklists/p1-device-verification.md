# P1 실기기 검증 체크리스트

> 대상: P1 완료 정의 ①②③ (docs/plans/p1-core-loop.md §4) | 2026-06-13 작성
> 환경: 실기기 iOS + Expo Go (Xcode 미설치 — dev build 불가), `npm run mobile`
> 전제: `.env`에 `EXPO_PUBLIC_OPENAI_API_KEY`(dev 전용). supabase 모드 검증은 로컬 스택
> (`supabase start` + `db reset`) 또는 호스팅 프로젝트 연결 후.

## 사용 방법

- 각 항목을 실기기에서 직접 수행하고 체크. 실패 항목은 **증상·재현 절차**를 아래 "결과 기록"에 적는다.
- 턴 지연 측정: Step 3 대화에서 **녹음 종료 탭 → Ted 음성 재생 시작**까지를 스톱워치로 5턴 이상 측정,
  중앙값 기록. 목표 **≤ 4s** (ADR-0003 기준 갱신용 — 측정값을 ADR에 반영한다).

## A. 레슨 3단계 음성 루프 (11항목)

- [ ] 1. 홈에서 오늘의 레슨 진입, Step 1 Learn 카드 표시
- [ ] 2. Learn: 표현 TTS 재생 (최초 — 네트워크 합성)
- [ ] 3. Learn: 같은 레슨 재진입 시 캐시 재생 (비행기 모드에서 재생 확인 — U4 완료 기준)
- [ ] 4. Drill: 녹음 → 전사 → 채점 동작 (통과 임계 80)
- [ ] 5. Drill: 오발음 시 누락 단어 하이라이트 + 모범 발음 재생, 2회 실패 후에도 진행 가능
- [ ] 6. 녹음 30초 cap: 30초 넘게 말하면 자동 종료·정상 처리 (U2)
- [ ] 7. Conversation: 4턴 대화 완주 — Ted 발화 문장 단위 순차 재생 확인
- [ ] 8. **턴 체감 지연 측정**: 5턴+ 중앙값 ______ s (목표 ≤4s)
- [ ] 9. 대화 중 교정 칩 inline 노출
- [ ] 10. 완료 화면: 잘한 점/개선점 요약, XP·streak 갱신, 홈 카드 완료 상태(일 1레슨 소프트 제한)
- [ ] 11. 레슨 중도 이탈 → 앱 재시작 → 이어하기 복원 (스냅샷 + 대화 턴 재조회)

## B. 폴백·엣지 (4항목)

- [ ] 1. 마이크 권한 거부 상태에서 텍스트 입력 폴백으로 Drill·Conversation 진행 가능
- [ ] 2. 비행기 모드에서 녹음 → 전송 실패 시 재시도 UX (로컬 녹음 보존)
- [ ] 3. 발화 10문장 이상 유도 확인 (부록 A — Learn 따라 말하기 + Drill + 대화 발화 합산)
- [ ] 4. 레슨 이탈 시 진행 중 AI 요청 취소 (이탈 직후 Ted 음성이 뒤늦게 재생되지 않음)

## C. 온보딩·로그인 — mock 모드 (Supabase env 미설정)

- [ ] 1. 첫 실행 → 로그인 화면 → "개발용 로그인" → 온보딩 4단계 → 홈
- [ ] 2. 마이크 권한 단계: 허용/거부 양쪽 모두 홈 진입 가능
- [ ] 3. 앱 재시작 후 온보딩 스킵·홈 직행 (로컬 persist)
- [ ] 4. 로그아웃 → 사용자 데이터 초기화 확인 (PII 정리)

## D. 온보딩·로그인 — supabase 모드 (env 설정)

- [ ] 1. 이메일 가입 → 온보딩 → profiles 반영 (goal/level/daily_goal_minutes/onboarded_at)
- [ ] 2. 레슨 완료 → user_progress 기록, 서버 streak 갱신
- [ ] 3. **로그아웃 → 재로그인 → 온보딩 스킵·홈 직행, 서버 streak 표시** (P1.5 V1)
- [ ] 4. 네트워크 차단 상태 재로그인 → 하이드레이트 실패해도 앱 진행 가능 (폴백)
